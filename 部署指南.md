# 音乐搜索网站 — 从零到上线完整指南

## 整体架构

```
你的域名（如 yinsou.com）
├── 前端  ← 已完成（gequhai/ 文件夹的 HTML）
└── 后端 API（NeteaseCloudMusicApi）  ← 部署在 Vercel（免费）
```

---

## 第一步：部署后端 API（NeteaseCloudMusicApi）

### 方式 A：Vercel 免费部署（推荐，不用服务器）

1. 注册 GitHub 账号（如果没有的话）
2. 打开 https://github.com/Binaryify/NeteaseCloudMusicApi ，点 Fork 到你自己的账号
3. 打开 https://vercel.com ，用 GitHub 登录
4. 点 "New Project" → "Import Git Repository" → 选你 Fork 的那个项目 → Import
5. 不用改任何设置，直接点 "Deploy"
6. 等 1-2 分钟部署完成，Vercel 会给你一个地址，类似：
   `https://neteasecloudmusicapi-xxx.vercel.app`

这就是你的 API 地址了，完全免费，自带 HTTPS。

### 方式 B：本地运行（学习测试用）

```bash
# 前提：电脑装了 Node.js（去 nodejs.org 下载安装）
git clone https://gitcode.com/gh_mirrors/ne/NeteaseCloudMusicApiBackup
cd NeteaseCloudMusicApiBackup
npm install
node app.js
```

启动后访问 `http://localhost:3000` 就是你的 API 了。

## 第二步：API 接口说明

部署好后，你的 API 地址假设是 `https://neteasecloudmusicapi-xxx.vercel.app`

以下是前端需要对接的核心接口：

### 搜索歌曲
```
GET /cloudsearch?keywords=周杰伦
```
返回：歌曲列表（歌名、歌手、歌曲ID、专辑封面等）

### 获取歌曲播放链接
```
GET /song/url/v1?id=33894312&level=standard
```
返回：MP3 播放 URL

### 获取歌词
```
GET /lyric?id=33894312
```
返回：LRC 格式歌词

### 获取歌曲详情
```
GET /song/detail?ids=33894312
```
返回：歌名、歌手、专辑、封面图、时长等

## 第三步：修改前端 HTML 对接你的 API

需要改的文件：`gequhai/pages/index.html`、`search.html`、`play.html`

### 3.1 首页 index.html — 搜索功能

找到搜索按钮的点击事件（AJAX 请求部分），把：
```javascript
url: "api/s",
data: { "keyword": keyword }
```
改成：
```javascript
url: "https://neteasecloudmusicapi-xxx.vercel.app/cloudsearch",
data: { "keywords": keyword }
```
搜索成功后，跳转逻辑也要改，因为返回数据格式不同。

### 3.2 搜索结果页 search.html — 展示搜索结果

需要改 AJAX 请求为调用你的 API，然后解析返回的 JSON 渲染表格。

### 3.3 歌曲详情页 play.html — 播放器 + 歌词

需要调用 3 个接口：
- `/song/url/v1?id=xxx` 获取播放地址
- `/lyric?id=xxx` 获取歌词
- `/song/detail?ids=xxx` 获取歌曲信息

## 第四步：部署前端

### 最简单的方式（Vercel）

1. 把 `gequhai/` 文件夹推到你的 GitHub 仓库
2. 在 Vercel 上再创建一个项目，指向这个仓库
3. Vercel 会自动帮你部署上线

### 其他方式
- 阿里云 OSS / 腾讯云 COS（静态网站托管）
- GitHub Pages（免费，但国内访问慢）
- 任何支持静态网站的主机商

---

## 第五步：绑域名（可选）

- 买一个域名（阿里云万网 / 腾讯云 DNSPod，.com 大概 60 元/年）
- 在域名解析里添加 CNAME 记录指向 Vercel
- 在 Vercel 设置里绑定自定义域名


---

## 注意事项

1. NeteaseCloudMusicApi 返回的数据格式和歌曲海不一样，前端 JS 需要做适配
2. 部署到线上后，API 地址换成你的 Vercel 地址
3. 这个 API 仅供学习使用，建议自己在 API 前面加一层缓存（Redis），减少请求频率
